Real-World Scenario

What this looks like in practice.

This is not theoretical. This pattern appears in active investigations — sanitized from production DFIR casework.

Sanitized from active investigation

A mid-level database administrator at a regulated financial institution begins running bulk SELECT queries against customer PII tables during off-hours. The queries are syntactically valid, use their legitimate credentials, and stay within their IAM permissions. Over six weeks, they extract 2.3 million customer records in batches sized to stay below alerting thresholds. Every EDR, SIEM, and CSPM tool in the stack reports green.

Without AEGIS: Detected 4 months later during a routine access review. 2.3M records exfiltrated. NYDFS 72-hour notification triggered retroactively. Estimated exposure: $11M+.

With AEGIS: Behavioral baseline flags anomalous query volume and temporal pattern within 72 hours of first deviation. Automated alert escalates to SOC with full query timeline, affected tables, and row-count trajectory. Containment within the same week.

Architecture

How AEGIS works — without touching the production path.

AEGIS operates as an API-native sidecar to the database engine. It observes query patterns, access metadata, and session behavior through the database platform's telemetry APIs — not by proxying or intercepting live queries. Zero latency impact on production workloads.

Application→ queries flow normally to database engine
Database Engine→ telemetry API emits access patterns, query metadata, session context
AEGIS Sidecar→ behavioral baselining · anomaly scoring · real-time alerting
Detection Engine→ multi-agent AI triage · MITRE ATT&CK mapping · forensic timeline
Output→ SOC alert · compliance evidence · automated containment via API
01 — BASELINE

Behavioral fingerprinting

AEGIS learns per-user and per-service-account query patterns: what tables they access, when, at what volume, with what query structures. Deviations scored against a rolling behavioral model.

02 — DETECT

Multi-signal anomaly detection

Credential misuse, off-hours access, unusual data volumes, bulk export patterns, privilege escalation sequences, and cross-table correlation anomalies — all scored in real time.

03 — RESPOND

Automated triage & containment

Multi-agent AI generates MITRE ATT&CK mappings, builds forensic timelines, and packages regulatory evidence. API-driven containment actions integrate with existing SOAR playbooks.
